Синтаксис домашка

Варіан №3 викликає помилку, для вирішення потрібно перенести другу команду в наступний рядок, або додати в кінці першої команди ( ; ).

Змінні

Варіант №1 викличе помилку так як команди написані в одному рядку без використання розділювальних символів, для вирішення потрібно перенести другу команду в наступний рядок, або додати в кінці першої команди ( ; ).

Варіант №2 ім'я змінної написано неправильно без використання lowelCamelCase, правильне написання: myAge.

Типи данних

Варіант №1 буде виводити у вигляді строки для того щоб змінна userAge виводилась коректно потрібно замінити подвійні лапки на зворотні.

Варіант№2 невірний, вернеться значення infinity для виведення NaN потрібно вираз ділити на число.

Варіант№3 невірний, вернеться тип данний undefined, так як null це спеціальне значення для визначення змінної як пустоти, і має бути присвоєне змінній.
